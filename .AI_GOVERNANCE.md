## üë§ For Human Contributors

### AI Governance Structure

All AI tools are governed by centralized, version-controlled instructions in `/.ai/`:

```
/.ai/
‚îú‚îÄ‚îÄ README.md         # Governance overview
‚îú‚îÄ‚îÄ SYSTEM.md         # Master authority (all tools inherit)
‚îú‚îÄ‚îÄ STYLE.md          # Brand voice, persuasion
‚îú‚îÄ‚îÄ DOMAIN.md         # TCNA/NJ HIC technical authority
‚îú‚îÄ‚îÄ COMPLIANCE.md     # Legal, regulatory, accessibility
‚îú‚îÄ‚îÄ OUTPUT_RULES.md   # Code quality, naming, formatting
‚îú‚îÄ‚îÄ COPILOT.md        # Copilot adapter
‚îú‚îÄ‚îÄ GPT.md            # ChatGPT adapter
‚îî‚îÄ‚îÄ CODEX.md          # Autonomous agent adapter
```

**Purpose:**  
Guarantee all AI-generated code and content is:
- Technically accurate (TCNA 2024, ANSI, NJ HIC)
- On-brand (‚Äúdetail-obsessed, anti-corner-cutter‚Äù)
- Legally compliant (NJ HIC, accessibility, consumer protection)
- High quality (semantic HTML, accessible, performant, linted)

### Best Practices

1. **Trust the governance:** AI tools auto-reference these files.
2. **Verify outputs:** AI must cite sources (e.g., "Per DOMAIN.md ¬ß2.2").
3. **Report issues:** Update the relevant `/.ai/` file if rules are violated.
4. **Centralize rules:** Add new rules to `/.ai/`, not scattered comments.

**If AI seems wrong:**
- Ask: ‚ÄúWhich instruction file are you following?‚Äù
- Check: Does `/.ai/SYSTEM.md` or domain file support this?
- Correct: Update `/.ai/` if clarification is needed.

### Updating Instructions

1. Edit the appropriate `/.ai/` file (STYLE, DOMAIN, COMPLIANCE, OUTPUT_RULES).
2. Commit with: `docs(ai): update [rule description]`
3. Changes apply immediately to all AI tools.

**Inheritance hierarchy:**
- `SYSTEM.md` = Master (overrides all)
- `COMPLIANCE.md` = Legal (overrides preferences)
- `DOMAIN.md`, `STYLE.md`, `OUTPUT_RULES.md` = Equal (different domains)
- Tool adapters = Thin wrappers (inherit above)

## üöÄ Quick Start Scenarios

### 1. Marketing Copy
- Reference **[`/.ai/STYLE.md`](.ai/STYLE.md)** for voice and persuasion.
- Use 48 Laws framework (e.g., Law 6: bold headlines, Law 33: pain points).
- Contrast ‚Äúproper method‚Äù vs. ‚Äúshortcut‚Äù (never name competitors).
- Validate technical accuracy with **[`/.ai/DOMAIN.md`](.ai/DOMAIN.md)**.
- Ensure legal compliance with **[`/.ai/COMPLIANCE.md`](.ai/COMPLIANCE.md)**.

### 2. Code Generation
- Follow **[`/.ai/OUTPUT_RULES.md`](.ai/OUTPUT_RULES.md)** for standards.
- Use semantic HTML (`<nav>`, `<article>`, not `<div>`).
- Reference design tokens (no hard-coded colors: `var(--color-primary)`).
- Add accessibility attributes (ARIA, alt text).
- Use correct file naming (`kebab-case.html`, `camelCase.js`).
- Pass HTMLHint, ESLint, and Jekyll build.

### 3. Technical Writing (Blog)
- Use **[`/.ai/DOMAIN.md`](.ai/DOMAIN.md)** for TCNA/ANSI standards.
- Apply precise terminology (‚ÄúC2-rated modified thinset‚Äù).
- Cite ANSI standards (‚ÄúPer ANSI A108.02, LFT requires 95% coverage‚Ä¶‚Äù).
- Apply voice from **[`/.ai/STYLE.md`](.ai/STYLE.md)**.
- Include disclaimers from **[`/.ai/COMPLIANCE.md`](.ai/COMPLIANCE.md)**.

### 4. Contract/Legal Content
- Reference **[`/.ai/COMPLIANCE.md`](.ai/COMPLIANCE.md)** first.
- Verify NJ HIC requirements (license #13VH10808800, 3-day rescission).
- Check payment restrictions (10% or $1,000 max deposit, whichever is less).
- Include all required disclosures.
- Add: ‚ÄúThis is not legal advice. Consult an attorney.‚Äù

## üîç Troubleshooting

### AI Generated Wrong Output
1. Ask: ‚ÄúWhat instruction file are you following?‚Äù
2. Verify the cited rule in `/.ai/`.
3. Update `/.ai/` to clarify if needed.
4. Provide feedback: ‚ÄúThat violates OUTPUT_RULES.md ¬ß2.1‚Ä¶‚Äù

### AI Seems Confused
1. Ask: ‚ÄúAre you referencing /.ai/SYSTEM.md?‚Äù
2. Provide context: ‚ÄúWe use design tokens from _tokens.scss.‚Äù
3. Cite rules: ‚ÄúPer OUTPUT_RULES.md ¬ß2.2, use var(--color-primary).‚Äù

### Instruction Conflicts
1. Hierarchy: SYSTEM.md > COMPLIANCE.md > others.
2. Legal overrides preferences.
3. Report or clarify in SYSTEM.md.

### Adding New Rules
1. Identify domain: Behavioral, technical, legal, code quality.
2. Edit the correct file.
3. Use clear sections and examples.
4. Commit: `docs(ai): add rule for [specific case]`
